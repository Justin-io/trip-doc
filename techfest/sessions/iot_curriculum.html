SESSION CURRICULUM: IoT DASHBOARDS & HARDWARE
Duration: 15 Minutes (Flash Talk) | Level: Engineering/Hardware
1. SESSION OBJECTIVE
To breakdown the "IoT Water Dispenser" project (Zone D) technically. Explaining how a button click on a website turns on a physical motor 10 meters away instantly.
2. AGENDA BREAKDOWN
Time Topic & Key Talking Points
0:00 - 3:00 The IoT Architecture (3 Layers)
1. Perception Layer (Sensors/Actuators - The Pump).
2. Network Layer (Transport - WiFi/MQTT).
3. Application Layer (Dashboard - The Web App).
3:00 - 8:00 Protocols: HTTP vs MQTT
- Why typical HTTP (Request/Response) is bad for real-time IoT.
- MQTT Pub/Sub Model: Broker, Publisher, Subscriber.
- Demo: Showing the HiveMQ dashboard receiving messages.
8:00 - 12:00 The Hardware Brain (ESP32)
- Why ESP32 over Arduino Uno? (Built-in WiFi/Bluetooth).
- GPIO Pins and Logical Levels (3.3V vs 5V).
- Controlling AC/High-Current DC loads using Relays (Opto-isolation).
12:00 - 15:00 Data Visualization
- How to plot sensor data in real-time (e.g., Temperature/Usage stats).
- Tools: Grafana, Node-RED, ThingsBoard.
3. KEY CONCEPTS TO COVER
  • Latency: The delay between command and action.
  • QoS (Quality of Service): MQTT levels 0, 1, 2.
  • Edge Computing: Processing data on the chip vs in the cloud.
4. STUDENT RESOURCES
  • Hardware: ESP32 Dev Kit V1, NodeMCU.
  • Platform: Arduino IDE, PlatformIO.
  • Cloud: Blynk, HiveMQ, AWS IoT Core.
