SESSION CURRICULUM: IoT DASHBOARDS & HARDWARE
Duration: 45 Minutes (Workshop & Demo) | Level: Engineering/Hardware
1. SESSION OBJECTIVE
To provide a comprehensive breakdown of the "IoT Water Dispenser" project. Students will learn the full stack of IoT: from wiring sensors and microcontrollers (ESP32) to standard protocols (MQTT) and cloud visualization (Dashboards).
2. AGENDA BREAKDOWN
Time Topic & Key Talking Points
0:00 - 10:00 Introduction to IoT Architecture
- Definition: Connecting physical objects to the digital world.
- The 4 Layers:
1. Perception (Sensors/Actuators - The "Skin" & "Muscles").
2. Network (Connectivity - WiFi, BLE, LoRaWAN).
3. Processing (Cloud/Edge - The "Brain").
4. Application (Dashboard/App - The Interface).
10:00 - 20:00 Deep Dive: Communication Protocols
- HTTP vs MQTT: Why standard Request/Response is too heavy/slow for IoT.
- MQTT (Message Queuing Telemetry Transport):
- Broker: The post office (HiveMQ/Mosquitto).
- Pub/Sub Model: Decoupling devices.
- Topics: Directory structure (e.g., `home/kitchen/temp`).
- QoS: Levels 0, 1, 2 (Fire-and-forget vs Guaranteed).
20:00 - 30:00 The Hardware Brain (ESP32) & Circuit Logic
- ESP32 vs Arduino Uno: Dual-core, built-in WiFi/BT, higher clock speed.
- Safety Logic: Why we need 3.3V logic.
- Relays: Using low-voltage signals to control high-voltage/current devices (like the Water Pump). Opto-isolation explained.
30:00 - 40:00 Live Code Walkthrough (The "Water Dispenser")
- Arduino/C++: Walkthrough of the code used in Zone D.
- Connection Sequence: `WiFi.begin()` -> `client.connect()` -> `client.subscribe()`.
- Callback Function: Handling incoming messages to trigger GPIO pins.
- Dashboarding: Showing data on ThingsBoard or Grafana.
40:00 - 45:00 Future Trends & Q&A
- Edge Computing: AI on the chip (TinyML).
- 5G & NB-IoT: Massive machine type communication.
- Q&A.
3. KEY CONCEPTS TO COVER
  • Latency: Critical in valid control systems.
  • Power Consumption: Deep Sleep modes for battery devices.
  • Security: TLS/SSL for MQTT, not sending plain text.
  • OTA (Over-The-Air) Updates: Updating firmware remotely.
4. STUDENT RESOURCES
  • Hardware: ESP32 Dev Kit, Raspberry Pi Zero W, Breadboards.
  • Software: VS Code + PlatformIO (Better than Arduino IDE), Node-RED (Visual flow coding).
  • Cloud: Blynk (Easy), AWS IoT Core (Pro), HiveMQ (Broker).
